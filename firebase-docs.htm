Guía de Configuración de Firebase Firestore para noticias.html

Esta guía explica cómo configurar correctamente Firebase Firestore para que funcione con la página noticias.html de EKUSFM, que utiliza Firestore para mostrar noticias, videos, programas y otras funciones dinámicas.

1. Crear un Proyecto en Firebase

  1. Accede a Firebase Console y crea un nuevo proyecto.
  2. Asigna un nombre al proyecto, por ejemplo ekusfm-web, y sigue los pasos para crear el proyecto.

2. Configurar Firestore

  1. En el panel del proyecto, selecciona Firestore Database y crea una base de datos en modo Producción o Prueba según prefieras.
  2. Define las siguientes colecciones y documentos para alimentar la página:

Colecciones y Documentos

  • noticias: Contendrá documentos con las noticias.
    • title (string): Título de la noticia.
    • content (string): Contenido o resumen de la noticia.
    • date (timestamp): Fecha de publicación.
  • top10videos: Videos de YouTube del top 10 canciones más escuchadas.
    • videoId (string): ID del video de YouTube.
    • title (string): Título del video.
    • rank (number): Posición en el top 10 (1 a 10).
  • programas: Información de los programas de radio.
    • name (string): Nombre del programa.
    • schedule (string): Horario del programa.
    • description (string): Descripción breve.
  • features: Lista de otras funciones geniales para mostrar.
    • name (string): Nombre de la función.
    • order (number): Orden para mostrar en la lista.

3. Configurar Reglas de Seguridad

Para permitir lectura pública (solo lectura), configura las reglas de Firestore así:

service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read: if true;
      allow write: if false;
    }
  }
}

4. Obtener la Configuración de Firebase para la Web

  1. En el panel de Firebase, ve a Configuración del proyecto > General.
  2. En la sección Tus apps, agrega una nueva app web si no existe.
  3. Copiar el objeto de configuración que contiene apiKey, authDomain, projectId, etc.
  4. Reemplaza el objeto firebaseConfig en noticias.html con esta configuración.
