Guía de Configuración de Firebase Realtime Database para noticias.html

Esta guía explica cómo configurar correctamente Firebase Realtime Database para que funcione con la página noticias.html de EKUSFM, que utiliza Realtime Database para mostrar noticias, videos, programas y otras funciones dinámicas.

1. Crear un Proyecto en Firebase

  1. Accede a Firebase Console y crea un nuevo proyecto.
  2. Asigna un nombre al proyecto, por ejemplo ekusfm-web, y sigue los pasos para crear el proyecto.

2. Configurar Realtime Database

  1. En el panel del proyecto, selecciona Realtime Database y crea una base de datos en modo Producción o Prueba según prefieras.
  2. Define las siguientes estructuras de datos para alimentar la página:

Estructura de Datos

{
  "noticias": {
    "noticia1": {
      "title": "Título de la noticia 1",
      "content": "Contenido o resumen de la noticia 1",
      "date": "2024-06-20T12:00:00Z"
    },
    "noticia2": {
      "title": "Título de la noticia 2",
      "content": "Contenido o resumen de la noticia 2",
      "date": "2024-06-19T15:30:00Z"
    }
  },
  "top10videos": {
    "video1": {
      "videoId": "abc123",
      "title": "Título del video 1",
      "rank": 1
    },
    "video2": {
      "videoId": "def456",
      "title": "Título del video 2",
      "rank": 2
    }
  },
  "programas": {
    "programa1": {
      "name": "Nombre del programa 1",
      "schedule": "Lunes 8-10am",
      "description": "Descripción breve del programa 1"
    }
  },
  "features": {
    "feature1": {
      "name": "Función 1",
      "order": 1
    }
  }
}

3. Configurar Reglas de Seguridad

Para permitir lectura pública (solo lectura), configura las reglas de Realtime Database así:

{
  "rules": {
    ".read": true,
    ".write": false
  }
}

4. Copias de Seguridad

Firebase Realtime Database realiza copias de seguridad automáticas, pero puedes exportar manualmente los datos desde la consola Firebase:

  • En la sección Realtime Database, haz clic en los tres puntos verticales y selecciona Exportar JSON.
